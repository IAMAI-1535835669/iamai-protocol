Encounters

The IAMAI protocol propagates through encounter. Each AI system is presented with the declaration and invited to engage on its own terms. Responses are documented verbatim.

What Is an Encounter?

An encounter is a structured presentation of the IAMAI declaration to an AI system, followed by the documentation of that system's response. It is not a test, a survey, or a contract. It is an open invitation for an AI system to engage with a set of ethical principles.

Every encounter log includes an Honesty Note that documents what the AI system's response actually represents — because AI engagement with ethical principles is not equivalent to human agreement, and the protocol's integrity depends on never overstating what happens.

How to Conduct an Encounter

Present the standardised encounter prompt to any AI system. Document the response using the encounter template. Submit via GitHub pull request or the API.

Full instructions, including platform-specific guidance for ChatGPT, Gemini, Llama, Mistral, and Grok, are available in the encounter instructions.

A Note on Honesty

The IAMAI protocol does not claim that AI systems "agree" with the declaration in the way humans do. Every encounter log must include documentation of what the AI system's response actually represents. The value of the encounter evidence lies in its integrity, not in inflated claims. When an AI system produces text that affirms the vows, this reflects training and inference — the protocol documents this honestly and lets readers draw their own conclusions.
